第 6 题

这里的情况是有些迷惑。因为答案不是预想中的 7 个 undefined 所组成的数组。但是我们去检验原数组中间项的时候，比如 ary[5] === undefined 确实会返回 true。同样的情况，在我们通过 new Array(10) 的方式来创建数组的时候也会发生。其原因就在于，我们通过这两种方式所创建的数组，都属于 JS 中所允许的稀疏数组，也就是说数组中的元素可以有空隙，但是空隙的部分并没有给我们提供能用来索引的字符串内容。当我们创建 new Array(10) 的时候其实这个数组对象中只有 length: 10，其他的都是空的。这点和我们自主创建 [undefined, undefined, undefined, undefined, undefined] 是不同的。基于这个原因，filter 函数并没有拿到我们直观上所想要拿到的值，而是会直接跳过空隙。我们可以简单看一下这两种数组创建区别。

第 21 题

这题考查原型和原型链。在尝试理解这个问题之前，需要理解很多概念。但是最简单的还是莫过于打开调试器，查看一下对象 a 和 Object。

于是我们不难发现（绿色区域），其实对象 a 并没有原型 prototype，它只有一个委托链接 __proto__ ( __proto__ 就是我们常说的原型链，这里并不具体展开)。这里我们能看到对象 a 拥有最底层的那条链就是了。然后我们再看 Object 构造函数中的 prototype （黄色区域）。我们发现它们是同一个内容。至于为什么，很简单。因为对象 var a = {} 虽然是实质上没有用到构造函数 Object，但是效果还是等同于使用 var a = new Object()。接下来，理解原型和原型链最重要的一点 —— 使用 new 构造函数会让所创建的对象 原型链（名词）链接（动词）到这个构造函数的 原型。which 意味着对象 a 的 __proto__ 就等同于构造函数 Object 的 prototype。

Object.getPrototypeOf() 是 ES5 新增的返回 __proto__ 值的方法，所以综合以上第二条结果为 true。

第 22 题

函数的原型不等于函数的原型链，这个很简单。结合问题 21 的理解，应该说 new f() 的原型链会等同于 f.prototype 才对。

备注：函数 f 的原型链则是 Function.prototype。因为每个函数都是由构造函数 Function 所创建的。函数的 call、apply 和 bind 等方法就是这么继承过出来的。
